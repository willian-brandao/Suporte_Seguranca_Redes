Objetivos

Neste laboratório, você documentará uma rede que é desconhecida para você.

  • Teste a conectividade de rede.
  • Compilar informações de endereçamento do host.
  • Acesse remotamente dispositivos de gateway padrão.
  • Documentar configurações de dispositivo de gateway padrão.
  • Descubra dispositivos na rede.
  • Desenhe a topologia de rede.

Histórico / Cenário

Seu empregador foi contratado para assumir a administração de uma rede corporativa porque o administrador de rede anterior deixou a empresa. A documentação da rede está ausente e precisa ser recriada. Seu trabalho é documentar os hosts e dispositivos de rede, incluindo todos os endereços de dispositivo e interconexões lógicas. Você acessará remotamente dispositivos de rede e usará a descoberta de rede para concluir uma tabela de dispositivos e desenhar a topologia de rede.

Esta é a parte I de uma série de atividades em duas partes. Você usará a documentação criada nesta atividade para orientá-lo aosolucionar problemas da rede na Parte II, Packet Tracer - Desafio de solução de problemas - usando a documentação para resolver problemas.

Ao investigar e documentar a topologia de rede, anote os problemas que você descobrir que não seguem as práticas ensinadas no currículo da CCNA.

Instruções

Parte 1: Testar Conectividade

Faça ping entre os PCs e o servidor de Internet para testar a rede. Todos os PCs devem poder executar ping um no outro e no servidor da Internet.

Parte 2: Descobrir as informações sobre a configuração do computador

Vá para o prompt de comando de cada PC e exiba as configurações de IP. Registre essas informações na tabela de documentação.

Parte 3: Descubra informações sobre os dispositivos gateway padrão

Conecte-se a cada dispositivo de gateway padrão usando o protocolo Telnet e registre informações sobre as interfaces que estão em uso na tabela. A senha do VTY é cisco e a senha EXEC privilegiada é class.

C:> telnet IP_address

Parte 4: Reconstruir a Topologia de Rede

Nesta parte da atividade, você continuará gravando informações sobre os dispositivos na rede na Tabela de Endereçamento. Além disso, você começará a diagramar a topologia de rede com base no que você pode descobrir sobre as interconexões do dispositivo.

Etapa 1: Acesse tabelas de roteamento em cada dispositivo de gateway.

Use as tabelas de roteamento em cada roteador para saber mais sobre a rede. Faça anotações de suas descobertas.

Etapa 2: Descobrir dispositivos que não sejam de gateway.

Use um protocolo de descoberta de rede para documentar dispositivos vizinhos. Registre suas descobertas na tabela de endereçamento. Neste ponto, você também deve ser capaz de começar a documentar interconexões de dispositivos.

Parte 5: Explore ainda mais as configurações e interconexões do dispositivo

Etapa 1: Acesse as configurações do dispositivo.

Conecte-se aos outros dispositivos na rede. Reúna informações sobre as configurações do dispositivo.

Etapa 2: Exibir Informações do Vizinho.

Use protocolos de descoberta para aumentar seu conhecimento sobre os dispositivos e topologias de rede.

Etapa 3: Conecte-se a outros dispositivos.

Exibir informações de configuração para os outros dispositivos na rede. Registre suas descobertas na tabela de dispositivos.

Agora você deve saber sobre todos os dispositivos e configurações de interface na rede. Todas as linhas da tabela devem conter informações do dispositivo. Use suas informações para reconstruir o máximo possível da topologia de rede.

Reflexão

Quais ferramentas ou comandos você achou mais úteis para documentar a rede?

Área de Resposta
As respostas variam. Os seguintes comandos de ferramentas foram usados: show ip route, show running-config, show cdp servers detail e telnet.

A segurança é uma preocupação nesta rede. Quais são as duas medidas que podem ser tomadas para tornar a rede mais segura?

Área de Resposta
Primeiro, o Telnet está em uso na rede. Isso deve ser desabilitado e SSH usado em seu lugar. Em segundo lugar, o CDP está sendo executado em todos os dispositivos de rede, incluindo o roteador Hub, que está conectado ao ISP. O CDP deve ser desabilitado pelo menos na interface Hub to ISP.

Você pode ter notado que algumas das práticas usadas para configurar os dispositivos de rede estão desatualizadas, ineficientes ou não seguras. Faça uma lista de quantas recomendações você tem sobre como os dispositivos devem ser reconfigurados para seguir as melhores práticas. Faça uma pesquisa na Internet para obter recomendações, se desejar.

Liste suas recomendações abaixo:

Área de Resposta
As respostas variam. Alguns exemplos são: ativar a segurança da porta nos switches, ativar o SSH para acesso remoto e usar senhas mais complexas.
